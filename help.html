Description

The DINet Viewer allows to visualise an image graph.
This platform is a proof-of-concept to support a paper presented in DAS 2016 conference [1].

A set of image is structured in a Relative Neighbourhood Graph [2] (RNG).
Using the platform, one can evaluate the relevance of the descriptors or the distances used to describe and compare the images.

In this demonstration the network of a 100 words subset of the Bentham dataset [3] is visualised.
The features used are BlockHOG and the distance is LDTW. Please refer to [1] for more details.

References:

  • [1] Frédéric Rayar, Tanmoy Mondal, Sabine Barrat, Fatma Bouali et Gilles Venturini. "Visual Analysis System for Features and Distances Qualitative Assessment: Application to Word Image Matching". In the 12th International Workshop on Document Image Analysis. (DAS), 2016, Santorini, Greece.
  • [2] Toussaint, G. T. (1980). The relative neighbourhood graph of a finite planar set. Pattern Recognition 12, 261-268.
  • [3] B. Gatos, G. Louloudis, T. Causer, K. Grint, V. Romero, J. A. Sanchez, A. H. Toselli, and E. Vidal, "Ground-truth production in the tranScriptorium project," in 11th IAPR International workshop on document analysis systems (DAS), 2014, pp. 237-241.

Data Import/Export ▲

One can load three types of files:

  1. Features file (euclidian distance is used to yield the RNG)
    • The feature input file is a plain text (*.txt) file.
    • One line correspond to the features vector of one image.
    • First column correspond to the filename of the image, with its extension.
    • If features are decimal value, the decimal separator is a dot ('.').
    • The feature are sparated with a tabulation ('\t').
    • No tabulation at the end of a line.
    • No empty line at the end of the file.
  2. Distances Matrix file
    • The feature input file is a plain text (*.txt) file.
    • First column correspond to the filename of the image, with its extension.
    • The rest of the columns correponds to the (dis)similarity matrix columns.
    • If values are decimal, the decimal separator is a dot ('.').
    • The values are sparated with a tabulation ('\t').
    • No tabulation at the end of a line.
    • No empty line at the end of the file.
  3. Graph file
    4. Either json or gexf graph format files.
    See ~/data/Wang/Wang_CLD_100.dinet.json and ~/data/Wang/Wang_CLD_100.dinet.gexf.

One can export the computed and/or the laid out graph in the json or gexf format.

The Clear graph button allow to properly clean the workspace vefore loading another data file.

Graph Manipulation ▲

Once the graph is displayed, it can be manipulated with common interactions:

  • Pan: left-clic on the canvas background and drag.
  • Zoom: use the mouse wheel on the graph canavs to (de)zoom.
  • Hover node: use the mouse to hover over a node.
    It will display the label of the node and a larger thumbnail of the node image on the bottom-right of the canvas.
  • Hover edge: use the mouse to hover over an edge.
    It will became black and display the label of the node.
    In the given example files, the label correspond to the distance between two images.
  • Select node: left-clic on a node to select it.
    Its information will appear on the right panel.
    A zoom will occur to focus on the selected node.
    A black border will appear on the selected node, and its neighbours will be highlighted with a halo around them.
  • Unfocus: left-clicl on the graph canvas background (without dragging).

Graph Layouts

Several layout algorithms can be used to lay out the graph:

  • Force Atlas 2
    Jacomy M, Venturini T, Heymann S, Bastian M (2014). "ForceAtlas2, a Continuous Graph Layout Algorithm for Handy Network Visualization Designed for the Gephi Software". PLoS ONE 9(6).PDF
  • Force-Link
    Extension of Force Atlas 2.     API
  • Fruchterman-Reingold
    Fruchterman, Thomas M. J.; Reingold, Edward M. (1991), "Graph Drawing by Force-Directed Placement", Software â€“ Practice & Experience (Wiley) 21 (11): 1129-1164. PDF
  • Stress Majorization
    Emden R. Gansner, Yehuda Koren, Stephen North. "Graph Drawing by Stress Majorization". Graph Drawing Volume 3383 of the series Lecture Notes in Computer Science pp 239-250.     PDF
  • Maxent-Stress
    Emden R. Gansner, Florham Park, Yifan Hu, Stephen North. "A Maxent-Stress Model for Graph Layout". IEEE Trans Vis Comput Graph. 2013 Jun;19(6):927-40.     PDF

Force Atlas 2 and Force-Link need to be stopped, otherwise these algorithms will keep running on the background.

Stress Majorization and Maxent-Stress are the best choices to have a good trade-off between the visual aesthetics on the 2D plane and the original feature space topology preservation.

Graph Controls ▲

Using the graph controls, one can:

Reset the graph. The default view is then displayed.
Show/Hide edges. Edges are displayed by default.
Show/Hide nodes as images/plain disk. In case of internal node, the first representative is displayed.
Diplay the help page.
Change the size of the nodes.
Filter the edges with regard to their weights.

Network Exploration ▲

There are three ways to navigate through the graph, and thus explore the image collection.

1. Navigating in the graph canvas

One can simply navigate in the graph drawn in the canvas by clicking its nodes of interest.

2. Navigating using neighbours list

When a node is selected, its informations are displayed on the information panel. The panel contains the neighbours of the node.

The list of neighbours allows one to go from the current node image to another which is similar.

Hence, one can navigate in the collection from image to image.

3. Navigating using the log

Each node visited is kept in a navigation logfile. The logfile can be found in the "Navigation log" tab.

One can use this logfile to go back to an already visited node clicking on it on the tab.

One can also use the navigation control buttons:

Go back to the first selected node in the navigation log.
Go back to the previous node in the navigation log.
Go to the next node in the navigation log.
Go to the last selected node in the navigation log.
Go to the last selected node (can be useful after unfocusing)

About ▲

The DINet Viewer platform has been developed by Frédéric RAYAR in the context of his PhD. (frederic.rayar {at} univ-tours {dot} fr)

The PhD takes place in the Computer Science Laboratory of Tours (University François-Rabelais of Tours), under the supervision of Dr. Sabine Barrat, Dr. Fatma Bouali and Pr. Gilles Venturini.

The PhD endeavour work focus on jointly study the indexing and the visualisation for browsing large dynamic image collections, under the following constraints:

  • handling medium to large image collections: the challenge is to handle real world data sets that can contains thousands to millions of images;
  • handling dynamic image collections: as images keep flooding in a continuous way, it becomes mandatory to build structure in a dynamic way: images shall be either added or edited if needed;
  • providing interactive visualisations: relevant visualisations - that are easy to interpret by the user - must be combined to interactions to allow the user to wander around the collection and make discoveries
